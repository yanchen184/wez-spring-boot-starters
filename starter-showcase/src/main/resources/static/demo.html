如何體驗差異？

關閉 starter：把 application.yml 中 common.log.enabled 改為 false，重新啟動後打 API 對比差異：
    ✅ 自動 --> / <-- log   ✅ traceId/spanId   ✅ password 遮罩   ✅ [SLOW] 標記   ✅ 統一錯誤回應

JPA Demo 體驗流程

1. 建立幾筆商品和訂單，觀察回應中的審計欄位（createdBy, createdDate）
2. 更新商品，觀察 lastModifiedBy / lastModifiedDate 自動變化
3. 軟刪除一筆訂單，然後分別查「未刪除」和「全部」，對比差異
4. 恢復刪除的訂單，確認 deleted 變回 false
5. H2 Console：http://localhost:8080/h2-console（JDBC URL: jdbc:h2:mem:showcase）

Redis Demo 體驗流程

1. 先用 JPA 建立一筆商品（POST /api/jpa-demo/products）
2. 用 Redis 快取查詢（GET /api/redis-demo/products/1）— 觀察 console 有 SQL log（cache miss）
3. 再查一次同一筆 — console 沒有 SQL（cache hit），回應更快
4. 清除快取（DELETE /api/redis-demo/products/1/cache）
5. 再查一次 — 又會打 DB（cache miss）
6. 用 KEYS * 觀察 Redis 裡存了什麼 key

Actuator + Observability 體驗流程

1. 先打幾個 API（任何區塊都行），產生一些流量
2. 查看 /actuator/health — 觀察 DB、Redis 健康狀態
3. 打 @Timed API 5~10 次，然後查 /actuator/metrics/showcase.timed.method — 看 COUNT、MAX、P50/P95/P99
4. 打 @Observed API — 觀察 console 的 traceId（metric + trace 同時產生）
5. 打「隨機成功/失敗」多次，查 /actuator/metrics/showcase.random.result — 按 status 分類
6. 查 /actuator/metrics/http.server.requests — 所有 HTTP 請求的自動統計（不用寫任何程式碼）
7. 查 /actuator/prometheus — Prometheus 格式輸出（可直接接 Grafana）
